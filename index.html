Unit 1 Lesson 1: The Nitty Gritty of the Web
Components of the Web

There are four main components commonly used to describe the "web". The most common point of access for an individual is their personal computer with a web browser(1), such as Chrome or Firefox. Most users will communicate their request through the Internet(2), which is the largest computer network, via an Internet protocol. The most common Internet protocol is HTTP(3), which is a protocol that allows the computer to speak with Servers(4). Servers are in essence computers which have been optimized to store and host large amounts of data which are regularly accessed.

Understanding The Web breaks it down fairly well in and easy to understand no nonsense explanation.

HTML

Hyper Text Markup Language makes up the bulk of online documents. Anything can exist on the web, however HTML will be the most prolific type of document you will find.

What HTML is made of:
1.-- Text content (what you see on the page)
2.-- Markup (what the content "looks" like)
3.-- References to other documents (images, videos)
4.-- Links to other pages

HTML Markup and Elements

The whole markup; opening tag, content, and closing tag, is referred to as an element. A body containing "nested"tags, or multiple tags inside of each other, is perfectly acceptable. A very simple and generic example would be:

<name>contents</name>

Some common examples of tags would be:

<em> - emphasis

<b> - bold

This is a great video demonstrating HTML Markup.

HTML Attributes

Attributes provide additional information about HTML elements such as defining an image, or establishing a link. This is typically written as a tag, then attribute, value of the attribute, "contents", then the closing tag as seen below.

<tag attr="value">contents</tag>

A common example of a "tag" would be the Anchor (which establishes a link) - <a>

A second example is the Image tag -<img>

<img src="url" alt="text">

Since images don't have any contents they are considered a "void tag", and thus don't have a closing tag.

A video demonstrating HTML Attributes.

Whitespace

An interesting fact about HTML is that the computer does not recognize "space" in it's traditional role. No matter how many "spaces" an individual may type into a line, the computer only codes it as 1 gap until the next character. Some common ways to reflect spacing, or dropping down to a new line include the break, and paragraph tags. There are essentially two forms of formatting, inline, and block.

The easiest way I found to remember this is that The block examples are structural elements, while the inline elements are text-based.

<br> - break gives you whitespace so lines don't run on together, since the break tag has no content, it is a void tag. It is considered an inline tag as it is text based and not structural.

<p>- paragraph is for block style of creating white space, and you can adjust height and width etc. Since there is content to block style, there are closing tags.

Unit 1 Lesson 2: HTML with Panache
HTML vs. CSS vs. DOM

Watch while the differences (and connections) between the 3 are explained.

HyperText Markup Language (HTML) is the standard markup language used to create web pages. Think of HTML as the structure and layout of the house you are living in. It defines where and how the page is laid out, and set up, similar to the layout of your house.

Cascading Style Sheets (CSS) is the most common language used for describing the look and formatting of a document written in a markup language. Think of CSS as the interior design of your house. Does your house have hardwood flooring or carpet? Do you have painted walls? Most of the "design elements" in your code are CSS.

DOM

Document Object Model (DOM) is a programming API for HTML and XML documents. It defines the logical structure of documents and the way a document is accessed and manipulated. DOM is the treelike structure seen when opening development tools, but it actually looks more like a forest, which can contain more than one tree. DOM does not require that things are laid out like a tree (or forest), nor does it specify relationships between the trees in any way.

Unit 1 Lesson 3: Adding CSS Style to HTML Structure
CSS Syntax and Rules

CSS syntax has a different set of rules than HTML. The most basic way to break it down is that the syntax is a reflection of three parts. The first is the HTML element the CSS will apply too, the second is the CSS style title (property), and the third is the value of the second property. Each HTML element can have multiple properties, and each property within that selector can have its own value. The syntax is contained within curly brackets, and seperated by a colon between the title and value. Multiple properties are separated by a semi colon. If a property has more than one value, use a comma to seperate them, and if it has more than one word, use qoutes.

Repetition

Practice may make perfect, but repetitition makes for lost productivity and hard to read code. One of the crowning achievments in increasing productivity through code writing is via the use of CSS. By writing a single block of CSS which can affect multiple HTML elements, we reduce the need for repeated code and enable for quick changes to the layout and structure of our pages!

Various Gem's and Tidbits Worth Remembering

Computers are "stupid", though they can preform amazing feats of calculation, they only do so because they were told to by the program written for them. In general, any time you get frustrated you need to remember that it is behaving the way it is simply because you told it to behave that way!
